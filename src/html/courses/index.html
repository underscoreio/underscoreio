Our courses

We offer a wide variety of Scala training courses and workshops, covering everything from fundamentals and best practices to advanced software arhitectures. Browse our course directory for more information on what we have on offer.

Many of our shorter courses are designed to work together in packages. We also routinely custom workshops to suit individual customers' needs. By combining the correct elements, we can create the perfect syllabus for your developers. Fill out an enquiry form to see what we can offer your organisation.

We believe that learning should never stop. All of our face-to-face training courses come with free ongoing access to our online training material.

Essential Scalaz

Level: Intermediate
Length: 1 to 2 days

This course is for experienced Scala developers who want to take the next step in engineering robust and scalable systems. The course focuses on four key abstractions in the Scalaz library: Functor, Monoid, Applicative, and Monad, and through a series of mini-projects shows how these abstractions apply across a broad range of practical problems.

Essential Play Web Services

Level: Beginner to intermediate
Length: 1 day

Play is a powerful, lightweight web framework that focuses on scalability and developer productivity.

This course concentrates on using Play to create scalabale RESTful web services. Write fast, non-blocking asynchronous code using futures and promises. Efficiently process and validate JSON and XML data. Integrate with third party web services using Play's WS client API. Implement your own streaming services using using websockets and Akka actors. Build your whole API using clean, testable development practices.

Play is a powerful, lightweight web framework that focuses on scalability and developer productivity.

Essential Distributed Systems

Level: Intermediate Java or Scala
Length: 2 days

Distributed systems introduce new challenges in building reliable, scalable software. This course teaches the core theoretical principles for constructing distributed systems and invites particpants to build their own systems using Scala or Java and a set of elemental tools. We will cover three approaches -- message queues, distributed caches, and eventual consistency -- and learn the benefits and drawbacks of each. Finally, in a review session, we will bring together all of the principles covered so far to build a non-trivial distributed system in its entirety.
